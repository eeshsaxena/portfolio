## Where it all began

Growing up I had two very different ambitions — one was to study an MBA and build something big in business, the other was to dive deep into technology. I wasn't sure how to reconcile the two, until one afternoon in 7th standard I opened a text editor and typed my first `<html>` tag.

I was obsessed with Harry Potter at the time (still am, honestly). And I had a wild idea: build a fan website for it. No tutorials, no YouTube — just the browser's View Source button and a lot of trial and error.

![Harry Potter fan website homepage I built in 7th standard — dark theme, Hogwarts castle, floating candles, golden fonts](/static/harry-potter-website-1.png)

## The first real project

The site had everything a 12-year-old would obsess over — a dark starry background, golden serif fonts, a navigation bar with links to Houses, Spells, Characters, and a Marauder's Map page that didn't actually do anything. But it *looked* magical, and that was enough.

I hand-coded every page. I learned about `<table>` layouts (yes, those), CSS floats, and eventually discovered JavaScript when I wanted the navigation to glow on hover. It was chaotic and beautiful.

![Spells page of the Harry Potter fan site — parchment texture, list of spells with descriptions, Hogwarts crest](/static/harry-potter-website-2.png)

## From fan sites to full-stack

After that first site, I never stopped building. Every new technology became a new spell to learn. Over the years I went from:

- Static HTML/CSS fan pages → React & Next.js SPAs
- Copy-pasted JavaScript → TypeScript, Node.js, Express
- Hobbyist projects → research internships at IIT & IIIT
- Competitive programming as a hobby → Specialist on Codeforces, Guardian on LeetCode

The MBA dream never fully left either — I still think about the intersection of technology and business every single day. But that afternoon in 7th standard made one thing clear: I was going to build things, not just study about them.

## This portfolio site

This portfolio is the latest iteration of that same impulse. It started as a way to showcase projects, but it's become a playground for experimenting with technology — Three.js, WebGL, Remix, CSS animations, and more.

The dual-mode design (3D view + 2D view) is a nod to that original fan site philosophy: make it functional, but make it *feel* like something. If you want to see what's under the hood, the [source is on GitHub](https://github.com/eeshsaxena).

## What's next

I'm currently working on Graph RAG pipelines, NLP corpora for low-resource Indian languages, and edge AI for cardiac monitoring. Each project is another `<html>` tag — a blank page that becomes something real.

If you're reading this and something resonates, feel free to [reach out](/contact). I'm always up for a good conversation about what we're building next.
